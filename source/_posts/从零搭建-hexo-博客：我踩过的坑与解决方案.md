历经几个小时的折腾，我的 \*\*Velo.x 极客空间\*\* 终于正式上线了！🎉

虽然结果很帅气，但这过程可真是“步步惊心”。为了防止以后忘了，也为了帮到同样折腾 Hexo 的朋友，我把今天遇到的几个\*\*“天坑”\*\*记录下来。



\## 🛠️ 坑位一：图片上传后不显示？



我明明在本地电脑建了 \`images\` 文件夹，但推送到 GitHub 后，背景图死活不显示，全是灰色的。



{% note warning %}

\*\*原因：\*\* GitHub 有个机制，\*\*空文件夹会自动被忽略\*\*。而且如果路径写错（比如 \`img\` 和 \`images\` 混用），Cloudflare 根本找不到图。

{% endnote %}



\*\*✅ 解决方案：\*\*

1. 统一文件夹名称，推荐直接用 \`source/images/\`。

2. \*\*最稳妥的方法\*\*：直接在 GitHub 网页版上传图片。

3. 如果想换背景图，不需要改代码，直接把新图重命名为 \`banner.jpg\`，覆盖上传，Cloudflare 会自动部署。



\---



\## 💻 坑位二：Git Pull 报错 "Connection Reset"



当我想把 GitHub 上的最新配置同步回本地电脑时，CMD 黑框框给我报了个大红错：

\`fatal: unable to access... Recv failure: Connection was reset\`



{% note danger %}

\*\*真相：\*\* 这不是电脑坏了，也不是代码错了，纯粹是 \*\*GitHub 的网络连接被墙阻断了\*\*。

{% endnote %}



\*\*✅ 解决方案：\*\*

\* \*\*佛系法\**：只要网页端正常，本地报错不影响博客访问，可以直接无视。

\* \*\*重试法\**：多输几次 \`git pull\`，有时候网络通一下就过了。

\* \*\*终极法\**：给 Git 配置代理（这个以后再折腾，小白先跳过）。



\---



\## 💬 坑位三：评论区死活不显示



我配置了 \*\*Utterances\*\* 评论系统，代码都没问题，但文章下面就是空空如也，连个框都没有。



\*\*原因竟然是：\*\* 我这个博客是 Fork 来的，\*\*GitHub 默认关闭了 Fork 仓库的 Issues（问题）功能\*\*！而评论系统是依赖 Issues 存数据的。



\*\*✅ 解决方案 (只需 3 步)：\*\*

1. 打开 GitHub 仓库的 \*\*Settings\*\* (设置)。

2. 找到 \*\*Features\*\* (功能) 区域。

3. 把 \*\*Issues\*\* 前面的钩打上！✅



{% note success %}

刷新页面，评论框立马就出来了！那一刻简直泪流满面。😭

{% endnote %}
